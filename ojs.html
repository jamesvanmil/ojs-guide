Pre-review

  • After submission, article is in "Unassigned" queue, with status "Awaiting Assignment"
  • Editor may view summary to review submission metadata
  • Editor may add additional supplementary files
  • Assigned default section; Editor may assign section
  • Editor may add section editor (via user search interface) or another editor (via user search interface) or self
  • Editor may reject and archive submission - tiggers email SUBMISSION UNSUITABLE

In Review

  • After assignment, article is in Review
  • Opportunity to upload a Review version - recommended to sanitize the document to remove identifying information, including metadata (with link to instructions fo Microsoft office)

Selecting a reviewer

  • Editor may assign one or more reviewers
  • May choose between free-form review and a pre-designed review form
  • May select due-date prior to sending request
  • Editor may trigger email REVIEW REQUEST or REVIEW REQUEST ATTACHED or REVIEW REQUEST ONECLICK (depending on attachment workflow selection), to ask reviewer to review

Reviewing

  • Reviewer may respond with system, or with email, depending on workflow (Emails: REVIEW CONFIRM or REVIEW DECLINE)
  • With system, reviewer will be able to download files and supplements, and complete a CI statement.
  • If using review form, that is accessible here. If not using review form, reviewer may fill out comments to editor and to author & editor.
  • Reviewer may also upload files.
  • Reviewer makes a recommendation and submits to editor (triggers REVIEW COMPLETE email)

Post-review

  • Editor may send an email to acknowledge the review (REVIEW ACK)
  • Editor may view content and decision of the review
  • Editor records decision and may notify author via email - this email may also import reviewer comments from forms (EDITOR DECISION ACCEPT, EDITOR DECISION DECLINE, EDITOR DECISION RESUBMIT, EDITOR DECISION REVISIONS)
  • May also edit and send author notification to reviewers

Revisions Required

  • For minor revisions
  • Author may upload revised file and Editor may make decisions without another round of review

Resubmit for review

  • For major revisions
  • Author may upload revised file and Editor submits file for an additional round of review
  • If the same reviewer is used, the emails are different: REVIEW REQUEST ATTACHED SUBSEQUENT, REVIEW REQUEST ONECLICK SUBSEQUENT, REVIEW REQUEST SUBSEQUENT

Accept submission

  • Editor can select version and send to copyediting (and upload a new version if needed)

Decline Submission

  • Submission will be archived after author is notified

Copyediting

  • If a Layout Editor is being assigned, the link to select will appear
  • Copyedit instructions from setup appear here
  • Depending on whether you have copyeditors, you can assign one here - if not the assigned editor will copyedit

Initial copyedit

  • Initiate to send email to copyeditor (no email if Editor is handling): COPYEDIT REQUEST
  • Once complete copyeditor can upload and send email COPYEDIT COMPLETE
  • Editor can acknowledge with COPYEDIT ACK

Author copyedit

  • Initiate to send email author: COPYEDIT AUTHOR REQUEST
  • Once complete author can upload on editing dashboard and send email COPYEDIT AUTHOR COMPLETE
  • Editor can acknowledge with COPYEDIT AUTHOR ACK

Final copyedit

  • Initiate to send email to copyeditor (no email if Editor is handling): COPYEDIT FINAL REQUEST
  • Once complete copyeditor can upload and send email COPYEDIT FINAL COMPLETE
  • Editor can acknowledge with COPYEDIT FINAL ACK

Scheduling

  • Use selection to choose which issue the article will be published in - issue management is handled elsewhere

Layout

  • If a Layout Editor is being assigned, the link to select will appear
  • The goal of the layout workflow is to get the galley version uploaded. The layout version is intended to be used as the source version for generating the galley version. The galley version will be published. If you're not using OJS for publication, you may still want to keep your final versions here for reference.
  • Layout templated can up uploaded to serve as a guide for layout editors and proofreaders

Layout Version

  • The Editor should manually upload the final copyedit version to the layout version
  • Layout version is intended to be used to retain the source document used to create the galley version - if there is no layout editor, this need not be used.

Galley Version

  • Galley version is the format presented to the reader - PDF, HTML, etc.
  • Uploading a Galley invokes the "Edit a Layout Galley" screen, where file format label and language can be specified, or the file replaced
  • Multiple galley files may be uploaded

Supplementary Files

  • Supplementary files can be uploaded, which invokes the same interface that teh author sees during the submission process
  • Multiple supplementary files may be uploaded

Proofreading

  • If a Proofreader is being assigned, the link to select will appear
  • The proofreading workflow is completely optional
  • Proofreading instructions appear here

Author

  • Initiate to send email request to author to proofread: PROOFREAD AUTHOR REQUEST
  • Author logs in to view proofreading instructions and access the proof - logging in notes date work is underway
  • Corrections are noted in the Proofreading Corrections comment section, not on the original document
  • Author marks as complete, which sends email: PROOFREAD AUTHOR COMPLETE
  • Editor may then acknowledge proofread with email: PROOFREAD AUTHOR ACK

Proofreader

  • No Emails sent if Editor is proofreading
  • Editor initiates to send email to proofreader: PROOFREAD REQUEST
  • Workflow as with Author, with emails: PROOFREAD COMPLETE, PROOFREAD ACK

Layout

  • No Emails sent if Editor is doing layout
  • Editor initiates to send email to Layout Editor: PROOFREAD LAYOUT REQUEST
  • Layout editor reviews corrections and creates new layout/galley versions
  • Layout editor notes complete with email: PROOFREAD LAYOUT COMPLETE
  • Editor acknowledges with email: PROOFREAD LAYOUT ACK
